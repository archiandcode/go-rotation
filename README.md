# Rotation Go

Веб-сервис для перераспределения материалов из XLSX-файлов между логинами. Приложение принимает Excel-файл, выполняет ротацию или выравнивание нагрузки, показывает прогресс обработки в реальном времени и отдает готовый XLSX-файл с заполненной колонкой `Закрепить` и отдельным листом итогов.

Интерфейс и сообщения приложения рассчитаны на русскоязычного пользователя.

## Возможности

- Загрузка файлов `.xlsx` через веб-интерфейс.
- Два режима обработки:
  - `Ротация` - полное перераспределение материалов по колонке `Закрепить`.
  - `Выравнивание` - частичное выравнивание текущего закрепления.
- Авторизация по паролю.
- Отображение прогресса через WebSocket.
- Подробный лог этапов обработки.
- Отмена выполняющейся задачи.
- История задач с возможностью скачать готовые результаты.
- Очистка истории и удаление сохраненных XLSX-результатов.
- Хранение истории и результатов на диске, чтобы они сохранялись между перезапусками контейнера.

## Быстрый запуск через Docker Compose

1. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

2. Укажите пароль для входа:

```env
APP_PASSWORD=your-strong-password
```

3. Запустите сервис:

```bash
docker compose up -d --build
```

4. Откройте приложение:

```text
http://localhost:8889
```

5. Войдите с паролем из `APP_PASSWORD`.

## Локальный запуск без Docker

По умолчанию сервер слушает порт `8080`, использует папку `data` и пароль `admin`, если `APP_PASSWORD` не задан.

```bash
APP_PASSWORD=admin DATA_DIR=data ADDR=:8080 go run ./cmd/server
```

После запуска откройте:

```text
http://localhost:8080
```

## Переменные окружения

| Переменная | Назначение | Значение по умолчанию |
| --- | --- | --- |
| `APP_PASSWORD` | Пароль для входа в веб-интерфейс | `admin` |
| `ADDR` | Адрес и порт HTTP-сервера | `:8080` |
| `DATA_DIR` | Каталог для истории и результатов | `data` |

В `docker-compose.yml` используются:

```yaml
environment:
  - ADDR=:8080
  - APP_PASSWORD=${APP_PASSWORD}
  - DATA_DIR=/app/data
volumes:
  - ./data:/app/data
ports:
  - "8889:8080"
```

Это значит, что внутри контейнера сервер работает на `8080`, а с хоста доступен на `8889`.

## Формат входного Excel-файла

Приложение работает с первым листом XLSX-файла. В первой строке должны быть заголовки. Обязательные колонки:

| Колонка | Для чего используется |
| --- | --- |
| `РП` | Группа/регион/портфель, внутри которого выполняется распределение |
| `ИИН` | Идентификатор должника; материалы группируются по ИИН |
| `Общая задолженность...` | Сумма задолженности; колонка ищется по началу названия `Общая задолженность` |
| `Открепить` | Исходный логин для режима ротации |
| `Закрепить` | Целевой логин; сюда записывается результат |
| `Статус` | Статус материала; часть статусов фиксируется и не перераспределяется |

Название колонки суммы может начинаться с `Общая задолженность`, например `Общая задолженность, KZT`.

### Парсинг сумм

Суммы читаются устойчиво к разным форматам:

- `1234567.89`
- `1 234 567,89`
- `1.234.567,89`
- `1,234,567.89`
- `5.6888806E+06`
- `5,6888806E+06`

Для расчетов используется `decimal`, поэтому арифметика не зависит от ошибок бинарного `float`.

## Режимы обработки

### Ротация

Режим `Ротация` отправляется на endpoint `/rotate/`.

Что делает:

- читает исходные логины из колонки `Открепить`;
- группирует строки по `РП + ИИН`;
- распределяет группы по логинам внутри РП;
- заполняет колонку `Закрепить`;
- формирует лист `Итоги ротации`.

Фиксированные статусы для ротации:

- `Оплата ПВ`
- `Оплата по соглашению`
- `Оплата по первому условию`
- `Оплата по второму условию`
- `Полное погашение`
- `Полное погошение`

Строки с этими статусами остаются закрепленными за текущим логином.

### Выравнивание

Режим `Выравнивание` отправляется на endpoint `/balance/`.

Что делает:

- читает текущее закрепление из колонки `Закрепить`;
- выполняет частичное выравнивание нагрузки внутри каждого РП;
- ограничивает объем перемещений;
- формирует лист `Итоги выравнивания`.

Дополнительный фиксированный статус для выравнивания:

- `Должник обещает оплату`

## Результат обработки

Готовый XLSX содержит:

- исходный первый лист с заполненной/обновленной колонкой `Закрепить`;
- отдельный лист итогов:
  - `Итоги ротации` для режима ротации;
  - `Итоги выравнивания` для режима выравнивания.

Итоговый лист содержит колонки:

| Колонка | Описание |
| --- | --- |
| `РП` | РП логина |
| `Логин` | Логин исполнителя |
| `Количество материалов` | Количество строк/материалов |
| `Сумма задолженности` | Суммарная задолженность |
| `Количество ИИН` | Количество уникальных ИИН |

Также внизу итогового листа добавляются служебные строки:

- количество строк, зафиксированных по статусам оплаты;
- количество ИИН, зафиксированных по статусам оплаты.

## Работа с интерфейсом

1. Откройте приложение в браузере.
2. Введите пароль.
3. Выберите режим: `Ротация` или `Выравнивание`.
4. Загрузите `.xlsx` файл.
5. Нажмите кнопку запуска.
6. Следите за прогрессом и подробным логом.
7. После завершения скачайте результат.

Во время обработки можно отменить задачу кнопкой `Отменить`.

## История задач

История отображается в нижней части интерфейса. Для каждой задачи видно:

- процесс;
- статус;
- процент выполнения;
- время запуска;
- время завершения;
- ссылка на скачивание, если результат готов.

История хранится в файле:

```text
DATA_DIR/history.json
```

Результаты сохраняются в:

```text
DATA_DIR/results/
```

При запуске через Docker Compose эти данные лежат в локальной папке:

```text
./data
```

Если сервер перезапустился во время обработки, задача, которая была в статусе `running`, помечается как отмененная с сообщением о прерывании при перезапуске.

## API

Все рабочие endpoints, кроме авторизации и проверки сессии, требуют активную cookie-сессию.

### Авторизация

`GET /api/me`

Возвращает статус авторизации:

```json
{
  "authenticated": true
}
```

`POST /api/login`

Тело:

```json
{
  "password": "your-password"
}
```

`POST /api/logout`

Завершает текущую сессию.

### Задачи

`POST /rotate/`

Запускает ротацию. Принимает `multipart/form-data`:

- `file` - XLSX-файл;
- `job_id` - идентификатор задачи, обычно генерируется клиентом.

`POST /balance/`

Запускает выравнивание. Формат запроса такой же.

`GET /api/jobs`

Возвращает список задач.

`GET /api/jobs/{job_id}`

Возвращает одну задачу вместе с логом прогресса.

`POST /api/jobs/{job_id}/cancel`

Отменяет задачу, если она еще выполняется.

`POST /api/history/clear`

Очищает историю и удаляет XLSX-файлы результатов.

`GET /download/{job_id}/`

Скачивает результат готовой задачи.

### WebSocket

`GET /ws/rotation/{job_id}/`

Используется для live-прогресса. Сервер отправляет JSON-сообщения:

```json
{
  "type": "progress",
  "percent": 48,
  "message": "Считаю нагрузку по логинам и подбираю распределение по РП"
}
```

При завершении:

```json
{
  "type": "job_ready",
  "message": "Файл готов.",
  "download_url": "/download/{job_id}/"
}
```

При ошибке:

```json
{
  "type": "job_error",
  "message": "Описание ошибки"
}
```

При отмене:

```json
{
  "type": "job_canceled",
  "message": "Задача отменена."
}
```

## Ограничения и поведение

- Одновременно выполняется только одна задача.
- Поддерживается только формат `.xlsx`.
- Обрабатывается первый лист книги.
- Заголовки должны находиться в первой строке.
- При отсутствии обязательных колонок обработка завершится ошибкой.
- Сессии авторизации хранятся в памяти процесса и действуют 24 часа.
- При перезапуске сервера сессии сбрасываются, история задач сохраняется на диске.

## Безопасность

- Обязательно задайте собственный `APP_PASSWORD` в `.env`.
- Не используйте пароль `admin` в рабочей среде.
- Если приложение публикуется наружу, запускайте его за reverse proxy с HTTPS.
- Файлы результатов сохраняются на диске в `DATA_DIR/results`, поэтому ограничьте доступ к директории данных на уровне сервера.

## Частые проблемы

### Не могу войти

Проверьте значение `APP_PASSWORD` в `.env` и перезапустите контейнер:

```bash
docker compose up -d --build
```

### Приложение пишет, что уже идет операция

Сервис допускает только одну выполняющуюся задачу. Дождитесь завершения текущей задачи или отмените ее.

### Файл не принимается

Проверьте, что:

- файл имеет расширение `.xlsx`;
- обязательные колонки есть на первом листе;
- заголовки находятся в первой строке.

### После перезапуска задача стала отмененной

Это ожидаемое поведение. Выполняющаяся задача не может быть продолжена после остановки процесса, поэтому при старте сервера она переводится в статус отмененной.
